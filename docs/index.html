Features

Everything you need, nothing you don't

Built for local AI workloads where VRAM is precious and containers stay loaded forever by default.

🎯

Transparent proxying

All requests pass through untouched — streaming, SSE, chunked responses, large bodies. Your AI clients need zero changes.

💤

Idle detection

Per-proxy configurable idle timeout. Stop the container after N seconds of silence, restart or unload on demand.

⚡

Cold-start on demand

Incoming request while container is stopped? Vigil starts it, waits for it to be healthy, then forwards — the client just sees a slow first response.

🖥️

Live web UI

See all containers, configure proxies, watch idle bars and live request logs. No config files, no YAML.

🔒

Concurrency control

Per-proxy max concurrency cap with a semaphore. Protect memory-hungry models from being overwhelmed in parallel.

📦

Persistent config

Proxy configs survive restarts via a mounted volume. Set it once, leave it running.

How it works

Simple by design

Point your client at Vigil instead of the model container. That's it.

1

Request arrives

Client sends request to Vigil on the configured subpath

2

Wake if needed

If container is stopped, Vigil starts it and waits for health check

3

Forward

Request proxied to backend, response streamed back to client

4

Count down

Idle timer resets. When it expires, container is stopped or restarted

Quick start

Up in one command

Requires Docker and access to /var/run/docker.sock.

docker run 
# pull and run
docker run -d \
  --network host \
  -v /var/run/docker.sock:/var/run/docker.sock \
  -v vigil-data:/data \
  ghcr.io/refolded/vigil:latest
docker-compose.yml 
services:
  vigil:
    image: ghcr.io/refolded/vigil:latest
    network_mode: host
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock
      - vigil-data:/data
    environment:
      STARTUP_TIMEOUT: "120"
    restart: unless-stopped

volumes:
  vigil-data:

Open http://localhost:8474 — add your first proxy from the web UI.

Environment variables

Variable Default Description
STARTUP_TIMEOUT 120 Seconds to wait for container to become ready after start
DATA_FILE /data/proxies.json Path where proxy configs are persisted
